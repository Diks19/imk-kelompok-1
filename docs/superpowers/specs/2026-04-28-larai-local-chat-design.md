# LarAI - Local AI Chat Interface Design Spec

## Overview

LarAI is a private, locally-hosted web application that provides a ChatGPT-like interface for interacting with large language models (via Ollama) and image generation models (via a local Stable Diffusion instance). The project emphasizes privacy, speed, and a familiar user experience.

## Architecture & Infrastructure

- **Backend Framework:** Laravel (PHP 8.3)
- **Frontend Stack:** React, Inertia.js, Tailwind CSS (v4)
- **Database:** MariaDB (default for Laravel Boost projects)
- **Real-time Communication:** Laravel Reverb (WebSockets) for streaming text and image generation status.
- **Queue System:** Laravel Horizon or default database queues for handling long-running image generation tasks.
- **AI Services:**
  - **Text/Vision:** Ollama (local HTTP API)
  - **Image Generation:** Local Stable Diffusion WebUI API (e.g., Automatic1111)

## Data Models (Core Entities)

1. **`Conversation` (Chat Session)**

   - `id`, `user_id` (foreign key)
   - `title` (string, generated by AI)
   - `created_at`, `updated_at`
2. **`Message`**

   - `id`, `conversation_id` (foreign key)
   - `role` (enum: 'user', 'assistant', 'system')
   - `content` (text)
   - `type` (enum: 'text', 'image_generation_prompt')
   - `status` (enum: 'sending', 'streaming', 'completed', 'error')
   - `created_at`, `updated_at`
3. **`Attachment` (Images)**

   - `id`, `message_id` (foreign key)
   - `path` (string, local storage path)
   - `type` (enum: 'uploaded_vision', 'generated_output')

## User Interface & Experience

### Layout

- **Sidebar (Persistent):** Left-hand sidebar displaying a grouped list of past conversations (e.g., Today, Yesterday, Previous 7 Days). Includes a "New Chat" button at the top.
- **Main Chat Area:** Centered layout within the right pane. Messages are displayed chronologically (user on the right or distinctly styled, AI on the left).
- **Input Area:** Sticky text area at the bottom of the main chat pane. Includes buttons for submitting text and uploading images for vision models.

### Key Interactions

1. **Creating a Chat:** Clicking "New Chat" opens a blank state.
2. **Streaming Text:** When a user sends a message, the backend connects to Ollama, and Laravel Reverb streams the response tokens back to the frontend in real-time, creating a "typing" effect.
3. **Auto-Titling:** After the *first* exchange (user message + AI response) in a new chat, a background job fires off a request to Ollama to generate a short title for the conversation, updating the sidebar automatically.
4. **Image Uploads (Vision):** Users can upload an image alongside their prompt. The frontend sends the base64 encoded image or file path to the backend, which forwards it to an Ollama multimodal model (like `llava`).
5. **Image Generation:** If the user requests an image (e.g., using a specific slash command like `/imagine` or natural language detection), the request is queued. The UI shows a loading state for that specific message. Once Stable Diffusion completes the image, Reverb pushes the image URL back to the frontend to replace the loading state.

## Error Handling & Edge Cases

- **Ollama/SD Unreachable:** The UI must handle connection timeouts gracefully, displaying a clear "AI service is offline" error message within the chat flow rather than a generic 500 page.
- **Generation Failures:** If image generation fails (e.g., CUDA out of memory on the local machine), the queued job fails, and Reverb pushes an error state to that specific message block in the UI.

## Testing Strategy

- **Pest PHP** for backend feature tests (simulating API calls to mock AI endpoints).
- **Laravel Dusk** (or similar browser testing) to verify the Reverb WebSocket streaming behavior and UI state updates.
